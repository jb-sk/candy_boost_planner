# 依存関係チェック ワークフロー

依存関係の脆弱性とアップデート可能なパッケージを確認し、対応方針を提示する。

---

## 手順

### 1. 脆弱性チェック（npm audit）
// turbo
```powershell
npm audit 2>&1
```

### 2. アップデート可能なパッケージ確認
// turbo
```powershell
npm outdated 2>&1
```

### 3. 結果の分析と選択肢の提示

以下の形式でレポートを作成：

```markdown
## 📊 依存関係チェックレポート

### 脆弱性

| パッケージ | 深刻度 | 修正方法 |
|-----------|--------|---------|
| (パッケージ名) | (low/moderate/high/critical) | (修正コマンドまたは説明) |

### アップデート可能

| パッケージ | 現在 | 最新 | タイプ |
|-----------|------|------|--------|
| (パッケージ名) | (現在ver) | (最新ver) | (dependencies/devDependencies) |

---

## 🎯 推奨アクション

### 選択肢

| 選択 | コマンド | リスク |
|-----|---------|--------|
| A) 安全な修正のみ | `npm audit fix` | 低 |
| B) パッチ更新のみ | `npm update` | 低 |
| C) メジャー含む全更新 | `npm audit fix --force` | 高（要テスト） |
| D) 現状維持 | なし | なし |

どれを実行しますか？
```

---

## 注意事項

- **メジャーアップデート**（破壊的変更）は必ず選択肢として提示
- 自動実行しない
- 更新後は `npm run dev` または `npm test` での動作確認を推奨
